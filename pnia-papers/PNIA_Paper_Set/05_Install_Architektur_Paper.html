3 · OpSec-Standards & .gitignore WA-02.4

Strikte Trennung von Code und Geheimnissen — Schlüssel/Keys nie ins Repository. Konsistent mit der „Integrität & Wahrheit“-Säule.

Secrets / RAM-only

API-Keys in Vault/Env-Vars, niemals im Code. Kryptographische Schlüssel werden strikt vom Code getrennt und ausschließlich im Systemspeicher (RAM) geladen — keine Persistenz auf Disk. Rotierbare Keys, least-privilege.

Schlüssel-Verwahrung

Treasury-Keys via HSM/Custody (Sygnum); Multisig für privilegierte Calls.

Audit-Trail

ISO-20022-konforme, unwiderrufliche Logs; jeder Payout nachverfolgbar.

4 · Async High-Performance Execution

Die Settlement-Bridges sind als asynchrone Event-Worker ausgelegt: ein Listener auf Chain-Events, eine non-blocking API-Schicht für Payouts — Ziel ist der ≤9-Sekunden-Pfad (TIPS).

Umsetzung mit Python/HTTPX (asynchroner HTTP-Client), um blockierende Netzwerk-Calls und Ladezeiten zu eliminieren.

# Muster: async Event-Loop mit HTTPX (illustrativ)
import httpx

async def run():
    async with httpx.AsyncClient() as client:
        async for event in master_token.events():   # non-blocking listen
            if is_settlement_trigger(event):
                await client.post(PAYOUT_URL, json=event.payload)   # ISO 20022
