## 6. SQL-Beispielschema für eine datenbankbasierte Bridge

```sql
-- Tabelle für Workspace-Entitäten
CREATE TABLE workspace_entities (
    id INTEGER PRIMARY KEY,
    source_file TEXT NOT NULL,
    entity_type TEXT CHECK(entity_type IN ('state','person','organization','code_system','theme')),
    name TEXT NOT NULL,
    iso3 TEXT,
    role TEXT,
    related_entity TEXT
);

-- Tabelle für staatliche / offizielle Access Points
CREATE TABLE access_points (
    id INTEGER PRIMARY KEY,
    entity_id INTEGER REFERENCES workspace_entities(id),
    access_type TEXT CHECK(access_type IN ('official_portal','legal_source','memorial','database','norm','intelligence','defense','archive')),
    url TEXT,
    legal_basis TEXT,
    validation_source TEXT
);

-- Beispielabfrage: Alle offiziellen Portale für Staaten im Workspace
SELECT e.name, e.iso3, a.url, a.access_type, a.legal_basis
FROM workspace_entities e
JOIN access_points a ON e.id = a.entity_id
WHERE e.entity_type = 'state' AND a.access_type = 'official_portal';
```

## 7. Fazit

Diese Bridge dient als verifizierbares Verbindungsstück zwischen:

- den **internationalen Code-Systemen** im Workspace (ISO, ITU, ICAO, NATO, UPU etc.) und deren Eigentümer-Organisationen,
- den **Staaten** im Workspace und den universellen UN-/CIA-/World-Bank-Access-Point-Mustern,
- den **verstorbenen Schlüsselpersonen** und den typischen Gedenk-/Archiv-Access-Point-Kategorien,
- den **strukturellen Themen** (Geheimhaltung, exterritoriale Archive, SAPs, Black Budget, isolierte Forschungsstädte) und den realen, juristisch validierten Entsprechungen (Vatikan-Archiv, Los Alamos, ISOO, DoD SAPCO).

Alle URLs sind offiziell und den jeweiligen Regierungen, UN-Organisationen oder internationalen Normungsgremien zugeordnet. Für eine vollständige Einzel-Erfassung aller 200 Staaten und sämtlicher Personen ist eine halbautomatisierte Erweiterung des SQL-Schemas empfohlen.
